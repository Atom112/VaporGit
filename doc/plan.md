# 开发计划

## 1. 建议的功能模块拆分

建议以“基础能力优先，渲染体验持续增强”的方式迭代：

### 1.1 仓库管理
- 打开本地仓库
- 最近仓库列表
- 仓库状态总览（分支、HEAD、工作区状态）

### 1.2 基础 Git 操作
- `status`、`add`、`restore`
- `commit`
- `branch`（创建、切换、删除）
- `fetch`、`pull`、`push`

### 1.3 提交树可视化
- 基于 DAG 的提交图布局
- 分支颜色与路径分层
- 节点悬停信息（作者、时间、提交信息）
- 点击提交联动右侧 Diff 面板

### 1.4 文件 Diff 视图
- 文件列表（新增/修改/删除/重命名）
- Side-by-side 与 Unified 模式切换
- 行级高亮与快速定位
- 二进制文件与超大文件降级策略

### 1.5 扩展能力
- Git stash 管理
- Rebase/Cherry-pick 可视化引导
- 子模块与 LFS 提示支持

---

## 2. 技术设计建议

### 前端（SolidJS）
- UI：组件化布局（仓库栏 / 提交树 / Diff 面板）
- 状态管理：按域拆分（repo、graph、diff、settings）
- 渲染：提交树建议使用 Canvas/SVG 混合或纯 Canvas 提升大仓性能

### 后端（Tauri + Rust）
- 使用 Tauri Command 作为前后端桥接
- 将 Git 操作封装为服务层，避免 UI 直接拼接命令
- 建议使用 `git2` 或调用系统 Git（二选一，建议优先 `git2`）

### 性能策略
- 日志分页加载（按提交批次增量）
- Diff 按需加载（点击文件时再请求内容）
- 大仓库缓存（提交元数据、文件变更摘要）

---

## 3. 里程碑规划

- **M1（可用基础版）**
  - 打开仓库
  - 查看状态
  - 暂存与提交
  - 基础提交列表 + 文件 Diff

- **M2（图形增强版）**
  - DAG 提交树
  - 分支/标签可视化
  - 提交详情联动 Diff

- **M3（进阶协作版）**
  - 远程同步（fetch/pull/push）
  - 冲突提示与辅助处理
  - 历史操作日志与设置中心

---

## 4. 后续开发建议

- 优先定义前后端接口：先固定 Command 输入输出结构，再做 UI。
- 先实现“可用”再“好看”：基础命令稳定后再强化提交树动画与视觉。
- 引入自动化校验：前端 lint + Rust fmt/clippy + 基础集成测试。
